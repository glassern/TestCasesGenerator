# 模块文件目录

## 📁 目录说明
本目录用于存放AI智能分割后的业务模块文件（仅适用于大文档）。

## 🔄 自动生成文件
**分析vs分割的区别**：
- **模块分析**：始终执行，识别文档结构和模块列表
- **物理分割**：仅当文档超过2000行时，生成独立的模块文件

当文档超过2000行时，AI产品分析师会自动将文档物理分割为多个模块文件：
- `module_01_模块名.md` - 第一个业务模块
- `module_02_模块名.md` - 第二个业务模块
- `module_XX_模块名.md` - 其他业务模块

## 📝 文件格式
每个模块文件包含：
- 模块名称和描述
- 相关业务功能详细说明
- 从原始文档中提取的完整内容

## 🎯 使用方式

### 大文档（>2000行）：
1. AI产品分析师分析文档结构并生成物理模块文件
2. AI测试生成师逐个读取模块文件
3. 为每个模块生成对应的测试用例

### 小文档（≤2000行）：
1. AI产品分析师分析文档结构，输出模块列表（不生成物理文件）
2. AI测试生成师基于模块列表，从原始md文档中按模块生成测试用例
3. 保持模块化思维，但使用原始文档作为数据源

## ⚠️ 注意事项
- 此目录中的文件由AI自动生成，请勿手动修改
- **无论文档大小，模块分析始终执行**
- 模块分割/分析严格按照文档中的原始顺序进行
- 分割后的模块保持内容的完整性和业务逻辑的连贯性
- 小文档虽不生成物理文件，但同样进行完整的模块结构分析
