# 历史测试用例目录

## 📁 目录说明
本目录用于存放历史版本的测试用例文件，便于版本管理和对比分析。

## 📂 文件组织
建议按照以下方式组织历史文件：
```
test_case/
├── v1.0/
│   ├── 产品名_测试用例_v1.0.xlsx
│   └── 分析报告_v1.0.md
├── v2.0/
│   ├── 产品名_测试用例_v2.0.xlsx
│   └── 分析报告_v2.0.md
└── archived/
    └── 过期的测试用例文件
```

## 🔄 使用建议
1. **版本归档**：每次生成新版本测试用例时，将旧版本移入此目录
2. **版本对比**：保留历史版本便于对比测试覆盖度的变化
3. **回滚参考**：需要时可以参考历史版本的测试思路

## 📊 文件命名规范
- Excel文件：`产品名_测试用例_vX.X_YYYYMMDD.xlsx`
- 报告文件：`分析报告_vX.X_YYYYMMDD.md`
- JSON文件：`测试数据_vX.X_YYYYMMDD.json`

## ⚠️ 清理建议
- 定期清理过期的测试用例文件
- 保留最近3个版本的完整文件
- 重要里程碑版本单独归档保存
